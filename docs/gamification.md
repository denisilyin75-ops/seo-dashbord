> Спецификация всех будущих фаз — в [`legacy-spec/GAMIFICATION.md`](legacy-spec/GAMIFICATION.md).

---

## Зачем это

SEO-индустрия устроена так, что результат действия видно через 3-12 месяцев. Написал статью → она индексируется месяц → через полгода набирает трафик → через год приносит первую продажу.

Этот разрыв **убивает мотивацию**. Люди бросают проект на 3-м месяце, когда фундамент уже заложен, но результата не видно.

**Решение Phase A:** показать **немедленную** цифру — как каждое действие влияет на капитализацию (оценочную стоимость) портфеля.

## Что ты видишь

### Виджет в шапке

```
💎 $8,940  +$0/24h
[█████░░░░░░░░░░░░░░░░░░░░░░]
```

Три вещи:
- **$8,940** — текущая оценочная стоимость всего портфеля (сумма по всем сайтам)
- **+$0/24h** — изменение за последние сутки (green/red)
- **прогресс-бар** — сколько % от цели (по умолчанию $50,000)

Кликни **👁** → виджет скрывается, остаётся только маленькая **💎** чтобы вернуть.

### Toast после действий

Когда создаёшь / редактируешь статью:

```
✨ Создано · 💎 +$15
💾 Сохранено · 💎 +$5
```

Суффикс «💎 +$N» показывает **предсказанный прирост** к капитализации этим действием.

## Откуда берутся цифры

Капитализация рассчитывается агентом **Site Valuation** (`server/services/agents/site-valuation.js`). Два режима:

- **Asset-based** (раняя стадия, <$50/мес profit): оценка как «актив» — возраст домена + контент + E-E-A-T + momentum + свежесть + перелинковка
- **Revenue-based** (стабильный profit ≥$500/мес): `multiple × monthly_profit` (базовый multiple 30)
- **Hybrid** (между порогами): среднее

Для toast'ов используются **те же значения** что формула добавляет при перерасчёте:

| Действие | Прирост |
|---|---|
| Опубликовать review | +$15 |
| Опубликовать comparison | +$25 |
| Опубликовать guide | +$10 |
| Опубликовать quiz | +$30 |
| Опубликовать tool/calculator | +$40 |
| Опубликовать category page | +$20 |
| Обновить опубликованную статью | +$5 |
| Создать/редактировать draft | +$2-3 |

**Принцип:** цифры реальные и заниженные. Это НЕ «motivational» $35-50 за обзор (как в оригинальной GAMIFICATION.md спеке), а то что формула действительно добавит при следующем запуске агента.

Почему это важно: если показать раздутые $50 за обзор, оператор пронаблюдает что «+$50 × 10 обзоров = +$500», но следующий запуск Site Valuation покажет +$150 — доверие сломано. Честные цифры = устойчивая мотивация.

## Где настраивается

**Settings → «🎮 Гамификация»** (`/settings`)

Три toggle'а:
1. **💎 Портфель в шапке** — показывать/скрывать виджет
2. **🎉 Toast «+$X к капитализации»** — показывать/скрывать суффикс в тостах
3. **🛠 Кастомные значения impact** — Phase B (пока заглушка)

**Важно:** когда виджет или тост скрыты, **backend продолжает считать** всё как обычно. Это чисто визуальный toggle. Включил → видишь актуальный snapshot, а не сброс.

Настройки хранятся в `user_prefs` на сервере — не в browser localStorage — поэтому переживают смену устройства / чистку кеша / перезагрузку контейнера.

## Что значит «zombie tier»

В asset-based оценке есть **penalty за нет-revenue**:
- −**20%** если сайт активен (есть момент-активность за 30 дней)
- −**40%** если сайт «zombie» (0 обновлений статей за 30 дней)

Это честно отражает рынок: покупатель актива платит за cash flow; без него даже за накопленный контент/E-E-A-T дают только половину «теоретической» оценки.

Как выйти из zombie:
- Опубликовать/обновить минимум 1 статью за последние 30 дней
- ИЛИ подключить revenue-источник и зафиксировать в `site_metrics` (GA4/affiliate API)

**Подводный камень:** если статьи не синхронизированы в SCC БД через `syncAllWp`, сайт выглядит zombie для агента даже если на WordPress много свежего контента. Нужно запускать sync регулярно (или через агент).

## Delta24h / Delta30d

Дельты сравнивают текущую капитализацию с baseline в узком окне:

- **Delta24h**: baseline ищется в [-3 дн, -1 дн]
- **Delta30d**: baseline ищется в [-45 дн, -30 дн]

Фильтрация по `methodology` — то есть baseline должен быть рассчитан той же версией формулы. После рекалибровки формулы старые значения НЕ используются как baseline (иначе получалось бы «−$15k за сутки» из-за смены константы, а не реального падения).

**На старте будет Delta = $0** пока агент `site_valuation` не запустится дважды подряд (это cron или ручной запуск через `/agents`).

## Phase A ≠ Phase B-E

**Что уже есть (Phase A):**
- ✅ Live Portfolio Value виджет в шапке с toggle
- ✅ Action impact toasts (+$N)
- ✅ Настройки в Settings

**Что придёт потом:**
- **Phase B (P2):** редактируемая таблица «Action → Impact $» в Settings
- **Phase C (P2):** XP / Levels / Streaks (ежедневные)
- **Phase D (P3):** Achievements + Daily Rings (Apple-Watch-style)
- **Phase E (P3):** AI coach + 8 «hidden metrics» (Topical Authority, Internal Link Density, Brand Strength и т.д. — требуют Ahrefs/SEMrush API)

Решение о конкретных фазах принимаем когда оператор (или я) запросит.

## FAQ

**Q: Валюация явно завышена/занижена. Можно поправить?**
Отредактируй константы в `server/services/agents/site-valuation.js` (`PER_ARTICLE_VALUE`, `ageImpact`, penalty rates). Бампни `methodology` в записи INSERT (строка `'v2.1_calibrated_2026-04-18'`) до `v2.2_...` — это гарантирует что delta24h не будет сравнивать через границу калибровки.

**Q: Я не хочу видеть цифры в шапке, но хочу чтобы они считались для отчёта.**
Открой Settings → выключи «💎 Портфель в шапке». Виджет скроется, но агент продолжит писать в `site_valuations` таблицу. Когда захочется — включи, увидишь актуальное.

**Q: Я хочу видеть цифры, но без toast'ов после каждой правки.**
В Settings выключи «🎉 Toast «+$X к капитализации»» — останется только хедер-виджет.

**Q: Сколько раз в день считается?**
Site Valuation агент — `@weekly` cron по умолчанию + ручной запуск через `/agents` кнопкой ▶. Виджет в шапке читает **последнюю** запись из `site_valuations`, поэтому отражает капитализацию на момент последнего прогона агента (не реал-тайм).

**Q: Можно добавить свою цель вместо $50,000?**
Да. `/agents` → `site_valuation` → config → `target_exit_valuation_usd` (по умолчанию 50000). Цифра прокидывается в виджет через `GET /api/portfolio/valuation`.
