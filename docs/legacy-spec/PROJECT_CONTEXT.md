# Project Context — SEO/Affiliate Portfolio

> Этот файл — бизнес-контекст проекта. Держать в корне проекта рядом с CLAUDE.md.
> Claude Code использует его для понимания "зачем" при любой новой задаче.

---

## Бизнес

- **Тип:** партнёрский агентный агрегатор на базе SEO-контентных сайтов
- **Ниша:** бытовая техника (кофемашины и расширение)
- **Рынки:** RU (popolkam.ru) + NL (koffie-expert.nl)
- **Монетизация:** CPA/CPS через Admitad, Яндекс.Маркет, Bol.com, Awin
- **Модель:** соло-владелец, максимальная автоматизация, минимум штата
- **Цель:** пассивный доход → масштабирование → экзит (продажа портфеля)

## Стек

- WordPress + REHub (тема) + WooCommerce
- Content Egg (агрегация цен) + WP All Import (импорт товаров)
- Claude AI (генерация контента: обзоры, сравнения)
- Claude Code (автоматизация, скрипты, доработки через child theme / custom plugins)
- GA4 + Search Console + Google Sheets (7 листов) + Looker Studio (дашборд)

## Типы контента

- **Review** — обзор одной модели
- **Comparison** — сравнение 2-5 моделей
- **Category** — каталог с фильтрами
- **Guide** — экспертный гайд
- **Quiz** — интерактивный подбор
- **Tool** — калькулятор / инструмент

## Ключевые KPI

- **RPM** = Revenue / Sessions × 1000
- **EPC** = Revenue / Affiliate Clicks
- **CTR** = Affiliate Clicks / Sessions
- **Sales CR** = Sales / Affiliate Clicks

## Принципы (всегда соблюдать)

1. **Максимальная автоматизация** — любую повторяющуюся задачу → в агенты/скрипты
2. **Минимум постоянного штата** — фрилансеры на разовые задачи
3. **Изменения только через child theme, custom plugin или WP-CLI** — никогда не править основную тему или плагины
4. **Не трогать core WordPress, REHub и сторонних плагинов** — только расширять через хуки и фильтры
5. **Каждый сайт = отдельный продаваемый актив** — никаких зависимостей между сайтами
6. **Решения должны масштабироваться на 3-5 сайтов** — если работает только для одного сайта, это плохое решение
7. **Всё должно повышать стоимость при продаже** — документация, процессы, чистый код, историчность данных

## Главный принцип при принятии решений

> Если действие не приближает к одному из трёх — не делать:

> 2. Снижение операционной нагрузки (больше автоматизации)
