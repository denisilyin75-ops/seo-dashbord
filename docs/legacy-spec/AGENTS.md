# AI Agents — Спецификация автономных агентов

> Документ описывает всех AI-агентов системы: что делают, когда запускаются,
> какие уставки им нужны, насколько критично их настраивать.
> Используется вместе с CLAUDE.md как reference для Claude Code.

---

## 0. Базовая концепция

**Что такое "агент"** — это автономный воркер, который:
- Запускается по расписанию (cron) или по триггеру (webhook/событие)
- Выполняет одну конкретную задачу
- Использует AI (Claude) для принятия решений
- Пишет результаты в БД + лог
- Может оповещать пользователя (Telegram/email) или создавать задачи в плане

**Где живут:** В n8n workflow-ах (визуально) или как cron jobs на Node.js сервере. Рекомендую **n8n** — удобнее менять расписание и логику без деплоя.

**Принцип:** Агенты не меняют контент сами — они **предлагают изменения** (создают задачи в плане) или **помечают флагами**. Финальное решение за человеком. Исключение — "тихие" агенты (обновление цен, sync метрик).

## 2. Детальное описание каждого агента

### 🔴 Агент 1: Content Freshness (Актуальность контента)

**Миссия:** Поиск устаревших статей, которые потеряли трафик или упоминают старые модели.

**Частота:** Раз в неделю (воскресенье 04:00)

**Что делает пошагово:**
1. Берёт все статьи со статусом `published`
2. Для каждой смотрит:
   - Возраст (дата последнего обновления)
   - Динамику трафика за последние 30 дней vs предыдущие 30
   - Упоминаемые модели товаров (парсит текст, ищет product IDs)
3. Claude AI анализирует и классифицирует:
   - `fresh` — всё ок, не трогать
   - `minor_update` — обновить цены, добавить 1-2 новинки
   - `major_update` — переписать разделы, много устаревшего
   - `rewrite` — полностью переделать
4. Создаёт задачи в `content_plan` с приоритетом по типу

**Уставки (важность изменения):**

| Параметр | Default | Важность | Когда менять |
|----------|---------|----------|--------------|
| `min_age_days` | 90 | 🔴 Высокая | В быстро меняющихся нишах (электроника) — 60 дней, в медленных (посуда) — 180 |
| `traffic_drop_threshold` | -25% | 🔴 Высокая | Если много false positive → -35%. Если пропускает падения → -15% |
| `min_sessions_to_check` | 100/мес | 🟡 Средняя | Чтобы не тратить токены на хвост. Низкотрафные сайты → 30 |
| `check_affiliate_products` | true | 🔴 Высокая | Критично для affiliate — устаревший товар = ноль комиссии |
| `auto_create_tasks` | true | 🟡 Средняя | Если отключить — только помечает флагом, не засоряет план |
| `max_tasks_per_run` | 10 | 🟡 Средняя | Чтобы не создать 50 задач за раз. Увеличивать по мере роста команды |

**Пример уведомления:**
```
🔄 Content Freshness — Еженедельный отчёт
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Проанализировано: 47 статей
Требует обновления: 8
  🔴 Major update: 3 (трафик упал >25%)
  🟡 Minor update: 5 (цены, новинки)
  ✅ Fresh: 39

Созданы задачи в план (топ-3):
  1. [HIGH] Обзор De'Longhi Magnifica S — трафик -42%, не упомянута LatteGo
  2. [HIGH] Топ-5 до 30000₽ — 2 товара сняты с продажи
  3. [MED] Как выбрать кофемашину — цены устарели
```

### 🟡 Агент 2: Editor (Редактор)

**Миссия:** Проверка качества текста — грамматика, структура, читаемость, SEO-оптимизация.

**Частота:** Раз в 2 недели для `published`, **сразу после публикации** для `draft→published`

**Что проверяет:**
1. **Грамматика и орфография** (через LanguageTool API + Claude review)
2. **Структура:**
   - Есть ли H2, H3, H4
   - Длина абзацев (не более 4-5 строк)
   - Есть ли списки, таблицы
3. **SEO:**
   - Ключевое слово в title, H1, первом абзаце
   - Плотность ключа (не переспам)
   - Мета-description есть и ≤160 симв
   - Alt-тексты у изображений
   - Внутренняя перелинковка (минимум 3 ссылки на другие статьи сайта)
4. **Читаемость** (Flesch score для RU/NL)
5. **Affiliate-гигиена:**
   - Есть ли CTA к покупке
   - Партнёрские ссылки с `rel="sponsored"`
   - Сравнительные таблицы с ссылками

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `min_word_count` | 1200 | 🟡 Средняя | Для reviews — 1500, для quiz — 300 |
| `max_paragraph_lines` | 5 | 🟢 Низкая | Читаемость. Не критично |
| `keyword_density_min` | 0.5% | 🟡 Средняя | |
| `keyword_density_max` | 2.5% | 🔴 Высокая | Выше — переспам, риск штрафа |
| `min_internal_links` | 3 | 🔴 Высокая | Критично для SEO и поведенческих |
| `require_meta_description` | true | 🔴 Высокая | Без этого проваливается CTR |
| `require_alt_tags` | true | 🟡 Средняя | +Accessibility, +SEO |
| `min_flesch_score` | 40 | 🟢 Низкая | Для специализированных ниш OK ниже |
| `auto_fix_minor` | false | 🔴 Высокая | **Осторожно!** Если true — правит опечатки сам. Рекомендую `false` |

**Результат работы:**
- Генерирует отчёт с оценкой 0-100 по каждой статье
- Создаёт задачу в план, если score < 70
- Помечает статью флагом `needs_edit = true`

### 🔴 Агент 3: Price Watchdog (Хранитель цен)

**Миссия:** Следить, что цены и наличие товаров в статьях актуальны. **Самый критичный агент** для affiliate.

**Частота:** Ежедневно (08:00 и 20:00) или чаще в hot-seasons

**Что делает:**
1. Парсит статьи, находит все упоминания товаров (через Content Egg product IDs или Google Shopping API)
2. Для каждого товара проверяет:
   - Актуальная цена
   - Наличие на складе
   - Рабочая партнёрская ссылка
3. Если разница в цене > X% — обновляет через WP REST API
4. Если товар OOS — заменяет на аналог (AI выбирает замену) и отправляет алерт

**Уставки:**

| Параметр | Default | Важность | Когда менять |
|----------|---------|----------|--------------|
| `price_change_threshold` | 5% | 🔴 Высокая | Слишком маленький → постоянные апдейты. Слишком большой → пропускаешь изменения. 5% — золотая середина |
| `auto_update_prices` | true | 🔴 Высокая | Оставить true, иначе смысла нет |
| `auto_replace_oos` | false | 🔴 Критическая | Осторожно! Рекомендую false — AI может подобрать неоптимальный аналог |
| `check_affiliate_links` | true | 🔴 Критическая | Битая ссылка = потеря дохода |
| `hot_season_frequency` | hourly | 🔴 Высокая | Black Friday, предновогодние — каждый час |
| `alert_on_oos_top_products` | true | 🔴 Высокая | Если товар в ТОП-3 выручки ушёл OOS — срочный алерт |

**Пример алерта:**
```
🚨 Price Watchdog — КРИТИЧНО
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ OOS: De'Longhi Magnifica S (ТОП-2 по выручке)
   Упоминается в 5 статьях
   Предложенный аналог: De'Longhi ECAM 22.110
   
   → Подтвердить замену? [Да] [Нет]

💰 Цены обновлены: 12 товаров
📉 Повышение цены >15%: Jura E8 (+18%)
```

### 🟢 Агент 4: Metrics Collector

**Миссия:** Ежедневный сбор метрик из GA4 и GSC, запись в БД.

**Частота:** Ежедневно (03:00 UTC)

**Что делает:**
1. GA4 API: sessions, pageviews, revenue, conversions по страницам
2. GSC API: impressions, clicks, position, CTR по запросам
3. Admitad/Awin API (если есть): партнёрские клики, продажи
4. Записывает снэпшот в `site_metrics` и `article_metrics`
5. Пересчитывает производные: RPM, EPC, CR

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `cron_schedule` | `0 3 * * *` | 🟢 Низкая | 03:00 — когда данные в GA4 уже consolidated |
| `lookback_days` | 2 | 🟡 Средняя | Backfill за 2 дня (GA4 может обновлять данные) |
| `retry_on_fail` | 3 | 🟡 Средняя | Google APIs иногда фейлят |
| `alert_on_zero_data` | true | 🔴 Высокая | Если 0 сессий — проверить, не сломался ли трекинг |

**Важно:** Этот агент — фундамент для всех остальных. Без метрик ни Content Freshness, ни Performance Analyst не работают.

### 🟡 Агент 5: Performance Analyst (Аналитик)

**Миссия:** Находить закономерности в данных — что работает, что нет.

**Частота:** Еженедельно (понедельник 09:00)

**Что анализирует:**
1. **Топ/антитоп статьи** по RPM, CR, времени на странице
2. **Паттерны успешных статей:** тип, длина, структура, ключи
3. **Слабые места:** высокий трафик + низкий CR, низкий CTR в SERP
4. **Тренды:** рост/падение по категориям, сезонность
5. **Возможности:** страницы с высоким impressions + низким CTR (улучшить title/meta)

**Результат:**
- Weekly report в Telegram/email
- Задачи в плане для проблемных статей
- Идеи для Content Planner

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `period_days` | 30 | 🟡 Средняя | 30 дней — стандарт. 90 для трендов |
| `comparison_period` | previous | 🟢 Низкая | `previous` или `year_ago` |
| `min_sessions_for_analysis` | 50/мес | 🟡 Средняя | Отсекать шум |
| `ctr_opportunity_threshold` | ctr < 3% AND position < 20 | 🔴 Высокая | Это "низко висящие фрукты" — быстрый рост |
| `generate_ai_insights` | true | 🟡 Средняя | Если выключить — только голые цифры |

### 🟡 Агент 6: SEO Doctor

**Миссия:** Техническая SEO-проверка сайта.

**Частота:** Раз в 2 недели

**Что проверяет:**
1. **Скорость:** Core Web Vitals (через PageSpeed Insights API)
2. **Индексация:** все ли страницы в Google (через GSC API)
3. **Sitemap:** актуален, отправлен
4. **robots.txt:** не блокирует важное
5. **HTTPS:** сертификат валиден, нет смешанного контента
6. **Структурированные данные:** Schema.org на product/review страницах
7. **404 / 500 ошибки** (из GSC Coverage report)
8. **Дубли тайтлов и meta-descriptions**
9. **Orphan pages** (страницы без внутренних ссылок)

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `lcp_threshold` | 2.5s | 🟡 Средняя | Google считает хорошим ≤2.5s |
| `cls_threshold` | 0.1 | 🟡 Средняя | |
| `alert_on_deindex` | true | 🔴 Критическая | Если важные страницы вылетели — срочно |
| `alert_on_ssl_expiry_days` | 14 | 🔴 Критическая | За 14 дней до истечения |
| `check_schema_on_reviews` | true | 🔴 Высокая | Review schema = rich snippets = +CTR |

### 🟡 Агент 7: Competitor Scout

**Миссия:** Мониторинг конкурентов и новинок в нише.

**Частота:** Еженедельно (среда 10:00)

**Что делает:**
1. Берёт список конкурентов (настраивается вручную или через SERP scraping)
2. Ищет у них новый контент за неделю
3. Проверяет новые продукты/модели в индустрии (через RSS feeds, news aggregators)
4. AI анализирует: стоит ли нам тоже писать об этом
5. Создаёт идеи для Content Planner

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `competitors_list` | [] | 🔴 Высокая | Без этого агент бесполезен |
| `industry_rss_feeds` | [] | 🟡 Средняя | Например: coffee news sites |
| `new_products_sources` | admitad_new | 🟡 Средняя | Новинки в партнёрке |
| `auto_suggest_content` | true | 🟢 Низкая | Создаёт идеи в плане |
| `max_suggestions_per_run` | 5 | 🟡 Средняя | Не перегружать план |

**Пример результата:**
```
🔍 Competitor Scout — Weekly
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
У конкурентов:
  • coffee-guide.ru: +3 статьи (Saeco, новый De'Longhi, гайд по эспрессо)
  • baristahub.ru: +2 (обзор Breville, сравнение молочников)

Новые модели на рынке:
  • De'Longhi Rivelia — выходит в мае
  • Jura Z10 Aluminium — новый цвет

Идеи для плана (созданы):
  1. Обзор De'Longhi Rivelia (high priority, ждём релиз)
  2. Гайд "Как работает эспрессо-помпа"
  3. Сравнение Breville vs De'Longhi (конкуренты хайпят)
```

### 🟢 Агент 8: Content Planner

**Миссия:** Генерация новых идей контента на основе данных.

**Частота:** Раз в 2 недели (или по запросу)

**Что делает:**
1. Собирает сигналы:
   - GSC: запросы с impressions, но без страниц под них
   - Performance Analyst: успешные паттерны
   - Competitor Scout: что пишут конкуренты
   - Trend Hunter: сезонные пики
2. Claude AI генерирует список идей с обоснованием
3. Создаёт пункты в `content_plan` со статусом `idea`

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `ideas_per_run` | 10 | 🟡 Средняя | Больше — больше выбор, но план перегружается |
| `auto_add_to_plan` | false | 🔴 Высокая | Если true — все идеи в план. Рекомендую false — показывать для утверждения |
| `content_type_mix` | balanced | 🟢 Низкая | balanced / review-heavy / comparison-heavy |
| `seasonal_boost` | true | 🟡 Средняя | Учитывать сезон (подарки к НГ, летние тренды) |

### 🔴 Агент 9: Link Checker

**Миссия:** Поиск битых ссылок (404, timeout, redirects).

**Частота:** Еженедельно (четверг 02:00)

**Что проверяет:**
1. Все внутренние ссылки во всех статьях
2. Все внешние ссылки (с ограничением по скорости — 1 req/sec)
3. Партнёрские ссылки (отдельная логика — см. Affiliate Auditor)
4. Изображения (не битые ли)

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `check_external` | true | 🟡 Средняя | Можно выключить, если много внешних ссылок |
| `timeout_seconds` | 15 | 🟡 Средняя | |
| `retry_failed` | 2 | 🟢 Низкая | Временные фейлы |
| `auto_fix_internal_redirects` | true | 🟡 Средняя | Обновляет 301 редиректы на финальные URL |
| `alert_threshold` | 5 | 🟡 Средняя | Алерт, если больше N битых |

### 🔴 Агент 10: Affiliate Auditor

**Миссия:** Ежедневная проверка ВСЕХ партнёрских ссылок. Битая ссылка = прямая потеря денег.

**Частота:** Ежедневно (06:00)

**Что проверяет:**
1. Все affiliate ссылки → follow redirect chain → проверить, что финальная страница != 404
2. Валидность партнёрского параметра (не обрезан ли tracking ID)
3. Товар на финальной странице — ещё продаётся
4. Соответствие ссылки и упоминаемого товара (не подменили ли товар на странице магазина)

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `check_tracking_params` | true | 🔴 Критическая | Без tracking — комиссии не будет |
| `follow_redirects_max` | 5 | 🟡 Средняя | |
| `alert_immediately_on_top_articles` | true | 🔴 Критическая | Для статей из ТОП-10 — алерт сразу, не ждать отчёта |
| `auto_replace_broken` | false | 🔴 Критическая | Рекомендую false — ручное решение |

### 🟢 Агент 11: Trend Hunter

**Миссия:** Ловить хайп, сезонные пики, новости.

**Частота:** Ежедневно (утро)

**Источники:**
1. Google Trends API по ключам ниши
2. Yandex Wordstat (для RU)
3. Reddit / специализированные форумы
4. YouTube trending в нише

**Что делает:**
1. Ищет резкие всплески интереса
2. AI определяет, стоит ли реагировать
3. Если да — срочная задача в план с high priority

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `spike_threshold` | 200% | 🟡 Средняя | Рост 2x от baseline = тренд |
| `min_absolute_volume` | 1000 | 🔴 Высокая | Чтобы не реагировать на рост в масштабах "с 2 до 10" |
| `regions` | RU, NL | 🔴 Высокая | Только целевые рынки |
| `react_speed` | 24h | 🟡 Средняя | Насколько быстро создать задачу |

### 🟡 Агент 12: Cannibalization Detector

**Миссия:** Найти статьи, которые конкурируют друг с другом за один поисковый запрос (это плохо для SEO).

**Частота:** Ежемесячно

**Что делает:**
1. Берёт все запросы из GSC
2. Для каждого смотрит, сколько страниц сайта ранжируются
3. Если >1 странницы конкурируют в топ-20 — cannibalization
4. AI предлагает решение: объединить, редирект, переориентировать

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `min_queries_with_overlap` | 3 | 🟡 Средняя | Сколько запросов должны пересекаться |
| `position_threshold` | top 20 | 🟡 Средняя | Рассматривать только видимые позиции |
| `auto_suggest_action` | true | 🟡 Средняя | AI предлагает: merge / redirect / repurpose |

---

### 🔴 Агент 13: Backup Agent

**Миссия:** Ежедневные бэкапы всего.

**Частота:** Ежедневно (01:00), retention 30 дней

**Что бэкапит:**
1. SQLite БД дашборда
2. Каждый WordPress: файлы + MySQL dump
3. Загружает на S3/Backblaze B2 или второй VPS

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `retention_days` | 30 | 🔴 Критическая | Меньше — риск потерь. Больше — расходы на хранение |
| `offsite_storage` | true | 🔴 Критическая | Обязательно на другой сервер! |
| `encryption` | true | 🔴 Высокая | GPG для бэкапов |
| `verify_restore_monthly` | true | 🔴 Высокая | Раз в месяц проверка восстановления — иначе бэкап может быть битым |

---

### 🔴 Агент 14: Alert Manager

**Миссия:** Маршрутизация уведомлений. Не "выполняет задачи", а "следит и кричит".

**Частота:** Real-time (event-driven)

**Триггеры:**
- Трафик упал > 30% за день
- Сайт недоступен (uptime check)
- SSL истекает
- Affiliate ссылка сломалась в ТОП-статье
- Деплой завершён/упал
- Большое изменение позиций в GSC

**Каналы:**
- Telegram (main) — для критичного
- Email — для отчётов
- Дашборд (toast) — для минорного

**Уставки:**

| Параметр | Default | Важность | Комментарий |
|----------|---------|----------|-------------|
| `telegram_bot_token` | — | 🔴 Критическая | Настроить |
| `quiet_hours` | 23:00-08:00 | 🟡 Средняя | Не спамить ночью (кроме критичного) |
| `dedupe_window_minutes` | 60 | 🔴 Высокая | Не дублировать алерт о том же за час |
| `critical_override_quiet` | true | 🔴 Высокая | Критичные идут даже ночью |

## 5. Правила безопасности

**Золотые правила:**

1. **Никогда не давать агенту право удалять данные** — только помечать флагами
2. **Auto-fix только для обратимых действий** (цены можно откатить, удалённый пост — нет)
3. **Всегда логировать** что агент сделал и почему
4. **Rate limiting для AI-агентов** — не выжирать бюджет Claude API
5. **Circuit breaker** — если агент 3 раза подряд упал, отключить его и алерт
6. **Dry-run режим** для тестирования новых уставок

- **Claude Sonnet** для аналитики и генерации идей
- Кэшировать контекст сайта (Prompt Caching API) — экономия 50-80%

## 7. Настройки по этапам зрелости портфеля

### Старт (0-1000 сессий/сайт)
- Всё руками или минимум агентов (только Backup, Metrics, Alerts)
- AI-агенты могут давать false positives на малых данных

### Рост (1-10k сессий)
- Включить Content Freshness, Price Watchdog, Editor
- Настроить индивидуально по нише (min_sessions_to_check=30)

### Зрелость (10k+ сессий)
- Все агенты активны
- Настроить строгие пороги (чувствительность выше)
- Добавить кастомных агентов под специфику ниши

### Портфель (5+ сайтов)
- Единый алертинг
- Cross-site Performance Analyst (сравнение между сайтами)
- Приоритизация ресурсов между сайтами

## 8. Что НЕ автоматизировать

Важно понимать границы:

❌ **Не автоматизировать:**
- Финальную публикацию статей (risk reputation)
- Удаление контента
- Изменение цен > 20% (возможен баг парсера)
- Замену партнёра (Admitad → Awin)
- Ответы на комментарии пользователей
- Решения о смене стратегии сайта

✅ **Автоматизировать смело:**
- Сбор данных и анализ
- Предложения (tasks в плане)
- Обновление цен в пределах нормы
- Бэкапы
- Проверку ссылок
- Технические проверки
