## 🟡 Infra ready но не активировано

### Deploy Wizard V2 host-worker

UI live, backend готов. Не хватает host-worker'а запущенного на VPS:

```bash
# На VPS:
export DEPLOY_WORKER_TOKEN=$(openssl rand -hex 32)
echo "DEPLOY_WORKER_TOKEN=$DEPLOY_WORKER_TOKEN" >> /opt/scc/.env.deploy-worker
echo "DEPLOY_WORKER_TOKEN=$DEPLOY_WORKER_TOKEN" >> /opt/scc/.env
cp /opt/scc/deploy/deploy-worker.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable deploy-worker --now

# Проверка:
systemctl status deploy-worker
docker compose restart scc  # подхватить новый DEPLOY_WORKER_TOKEN
```

После этого Deploy Wizard V2 в UI (Dashboard → Deploy) будет реально запускать `provision-site.sh` на хосте.

### Per-site LLM budget UI

Backend принимает `monthlyLlmBudgetUsd` через PUT /api/sites/:id. UI поля пока нет (можно добавить через curl или API). Если приоритетно — попроси в след. сессии, 15 мин работы.
