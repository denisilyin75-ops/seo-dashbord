## 3. Pillar/cluster архитектура

```
PILLAR 1: Как выбрать беговые кроссовки (cornerstone-guide, главный hub)
  ├── cluster: пронация (3 sub-guides)
  ├── cluster: дроп / стеклоплёнка / стек (3 sub-guides)
  ├── cluster: размер и fit (2 guides)
  └── cluster: lifecycle — когда менять, как ухаживать (2 guides)

PILLAR 2: Top-rankings (10 hub-страниц)
  ├── По бренду: Asics, Nike, Saucony, Adidas — топ-X 2026 каждого
  ├── По задаче: марафон, трейл, тренировки, восстановление
  └── По бюджету: до 8k, 8–15k, премиум >15k

PILLAR 3: Сравнения (10 vs-страниц)
  ├── Flagship vs flagship: Pegasus 41 vs Cumulus 26 vs Adidas Adizero
  ├── Эволюция: «Asics Cumulus 23 → 26 — стоит ли апгрейд»
  └── Cross-brand: «Hoka Clifton vs Brooks Glycerin vs NB 1080»

PILLAR 4: Сезонные подборки (4 страницы/год)
  └── Зима / весна / лето / осень — обновляется каждый сезон

PILLAR 5: По профилю бегуна (5 страниц)
  └── Начинающий, женщина, тяжёлый (>90кг), асфальт vs трейл, ультра
```

Каждая страница топа/сравнения/гида = **внутренняя ссылка в 5–15 наших обзоров** + 1–2 ссылки на партнёрские карточки в магазинах. Цель — собирать low-frequency intents, направлять трафик в обзоры (где партнёрские ссылки), и перекрёстно прокачивать domain authority.

## 5. Внутренняя перелинковка (правило)

Каждая новая страница топа/сравнения/гида **обязана** ссылаться на:
- 5–15 наших обзоров (по теме топа) — `<a href="/obzor-krossovok-...">`
- 2–3 другие страницы плана (например, top для марафона ссылается на «как выбрать», на comparison Pegasus vs Cumulus, на гид по дропу)

Без этой перелинковки топ-страница = висит в воздухе и не качает обзоры. С перелинковкой = качает domain authority всем 343 обзорам.
